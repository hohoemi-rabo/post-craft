# Post Craft Phase 3 構想メモ

> 作成日: 2026-01-27
> ステータス: 構想段階（未実装）
> 前提: Phase 2 完了

## 1. Phase 3 コンセプト

**目標:** 現在の「自分専用カスタマイズ」から「ユーザーごとのカスタマイズ」へ拡張し、他のユーザーも自分のブランドに合わせた投稿素材を生成できるサービスにする。

### 現状（Phase 2）
- 投稿タイプ（6種類）がコードにハードコード
- 画像スタイル（4種類）がコードにハードコード
- テンプレート構造がコードにハードコード
- 必須ハッシュタグがコードにハードコード
- → すべて「ほほ笑みラボ」仕様

### Phase 3 構想
- ユーザーごとに投稿タイプを作成・編集可能
- ユーザーごとに画像スタイルを設定可能
- ユーザーごとにテンプレート構造を定義可能
- ユーザーごとに必須ハッシュタグを設定可能
- → 各ユーザーが「自分ブランド」の投稿素材を生成

## 2. 想定機能

### 2.1 投稿タイプ管理
- 投稿タイプの新規作成
- テンプレート構造の編集（プレースホルダー定義）
- 必須フィールド・任意フィールドの設定
- プレビュー機能

### 2.2 画像スタイル管理
- 画像スタイルの新規作成
- ベースプロンプトの編集
- キャラクター対応の設定
- サンプル画像の登録

### 2.3 ハッシュタグ管理
- 必須ハッシュタグの設定
- 推奨ハッシュタグリストの管理
- カテゴリ別ハッシュタグ

### 2.4 キャラクター管理（既存機能の拡張）
- 複数キャラクター対応（既存）
- デフォルトキャラクター設定（既存）
- キャラクター別プロンプト設定

## 3. 技術的な検討事項

### 3.1 データベース拡張

```
現在ハードコードされているもの → DBテーブル化

post_types テーブル
- id, user_id, name, icon, description
- template_structure, required_fields, optional_fields
- hashtag_trend

image_styles テーブル
- id, user_id, name, icon, description
- base_prompt, supports_character

user_settings テーブル
- id, user_id
- mandatory_hashtags
- default_image_style
- default_post_type
```

### 3.2 管理画面

新規ページ:
- `/settings/post-types` - 投稿タイプ管理
- `/settings/image-styles` - 画像スタイル管理
- `/settings/hashtags` - ハッシュタグ管理
- `/settings/templates` - テンプレート編集

### 3.3 課題・検討事項

1. **プロンプトエンジニアリングの難しさ**
   - ユーザーが自由にプロンプトを書くと品質がばらつく
   - テンプレート形式で簡単に設定できる仕組みが必要

2. **UIの複雑さ**
   - テンプレート構造の編集UI
   - プレースホルダーの設定UI
   - プレビュー機能

3. **料金プラン**
   - カスタマイズ機能は有料プラン限定？
   - 投稿タイプ数の制限？

## 5. 差別化ポイント（仮説）

1. **メモ書きからの自動生成**
   - 構造化されたテンプレートへの自動変換
   - 他サービスは手動入力が多い

2. **画像の同時生成**
   - キャプションと画像をセットで生成
   - キャラクター一貫性

3. **個人事業主・スモールビジネス向け**
   - 自分ブランドのテンプレート化
   - 投稿の統一感
