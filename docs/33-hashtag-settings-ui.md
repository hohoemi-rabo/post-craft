# チケット #33: ハッシュタグ設定 UI

> Phase 3 | 優先度: 高 | 依存: #27, #28, #29

## 概要

必須ハッシュタグの設定画面（`/settings/hashtags`）を実装する。

SPEC-PHASE3.md セクション 8.5 に準拠。

## 対象ファイル

| ファイル | 操作 |
|---------|------|
| `src/app/(dashboard)/settings/hashtags/page.tsx` | 新規作成 |
| `src/components/settings/hashtag-settings.tsx` | 新規作成 |

## 画面構成

### 必須ハッシュタグセクション

- タイトルと説明文
- 現在のハッシュタグ一覧（各タグに [× 削除] ボタン）
- 追加フォーム（テキスト入力 + [追加] ボタン）
- `#` プレフィックスの自動付与
- 残り追加可能数の表示（「残りN個追加可能」）
- 最大4個の制限

### 生成情報セクション

- 合計ハッシュタグ数の表示: `📊 合計: 10個（必須N個 + 自動生成M個）`
- 生成ロジックの説明テーブル:
  - 必須0個 → 自動生成10個
  - 必須1個 → 自動生成9個
  - 必須2個 → 自動生成8個
  - 必須3個 → 自動生成7個
  - 必須4個 → 自動生成6個

### 保存ボタン

- 変更がある場合のみ有効化
- 保存成功時にToast通知

## 受入条件

- ハッシュタグの追加・削除が動作する
- 4個を超えて追加できない
- 空のハッシュタグは追加できない
- `#` 付き/なしどちらで入力しても正しく処理される
- 生成数が動的に更新される（必須数に連動）
- 保存が正常に完了し、Toast通知が表示される
- 変更なしの場合は保存ボタンが無効
